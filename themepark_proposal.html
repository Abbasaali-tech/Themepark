3. Problem Statement

Theme parks handle many daily operations such as visitor registration, ride management, event scheduling, ticket booking, staff management, feedback collection, and revenue tracking. Manual handling of these tasks can lead to data inconsistency, poor record keeping, and difficulty in managing the park efficiently.

An object-oriented software solution is needed that models real-world park entities as classes with clear relationships, enforces business rules through encapsulation, and stores data reliably using file handling.

4. Proposed Solution

We propose to build a Theme Park Management System in C++ that fully utilizes OOP concepts taught in the lab. The application will run as a menu-driven console program with clearly separated classes for each entity.

Key Classes & Modules:

  • ParkEntity (Base Class): Abstract base with ID and name — inherited by Visitor, Ride, Event, Staff, and Inventory.
  • Visitor & Ride: Visitor stores age and email; Ride stores price and operational status.
  • Event: Stores event name, date, capacity, and number of registered visitors.
  • Staff: Stores staff name, role, and salary.
  • Ticket: Records ticket booking, supports discount and output formatting.
  • Feedback: Stores visitor feedback and rating.
  • Inventory: Tracks items, quantity, and cost per item.
  • User: Handles login credentials and role-based access.
  • ThemeParkSystem: Central manager class that handles login, menus, file loading, and file saving.

5. OOP Concepts to be Implemented

OOP Concept Where Applied
Classes & Objects Visitor, Ride, Event, Staff, Ticket, Feedback, Inventory, User, ThemeParkSystem
Encapsulation Private data members with getters and controlled access in all classes
Inheritance ParkEntity → Visitor, Ride, Event, Staff, Inventory
Polymorphism Virtual display() and serialize() overridden in derived classes
Abstract Classes ParkEntity as the abstract base class with pure virtual methods
Operator Overloading Ticket class overloads +, <, and <<
Templates Generic helper functions readNumber() and findById()
Exception Handling runtime_error and invalid_argument used for invalid operations
File Handling All records saved and loaded using text files
STL Containers vector, algorithm, numeric, stringstream used throughout
Friend Functions applyDiscount() and operator<< for Ticket, calculateAverageRating() for Feedback
Static Members Static nextId values used for automatic ID generation

1. Executive Summary

This report presents the complete design and implementation of the Theme Park Management System, a C++ console application developed for the Object Oriented Programming (OOP) Lab at FAST National University of Computer and Emerging Sciences, Karachi, under the supervision of Sir Muhammad Khalid Khan.

The system models a real-world theme park using a rich class hierarchy. It demonstrates all fundamental and advanced OOP concepts including encapsulation, inheritance, polymorphism, abstract classes, operator overloading, templates, exception handling, and file I/O. The project was developed by two members, Mohammad Sarim Raza (25K-0075) and Abbas Ali Haider (25K-0009). The final application allows staff to register visitors and book tickets, while the manager can add rides, events, staff, inventory, view feedback, and generate revenue reports — all persisted to files across sessions.

2. Introduction

2.1 Background & Motivation

Theme parks involve many daily operations such as visitor registration, ride management, event scheduling, ticket booking, staff management, feedback collection, inventory control, and revenue tracking. Handling these operations manually can lead to data inconsistency, poor record keeping, and difficulty in managing the park efficiently. This project was created to model these operations using Object Oriented Programming concepts in C++.

This project also serves as a practical exercise in applying all OOP concepts studied during the semester in a single application.

2.2 Problem Statement

The core problems addressed by this project are:

  • No automated way to track visitors, rides, events, and tickets.
  • Manual management of staff, inventory, and feedback records.
  • Revenue calculation is difficult without a structured software system.
  • Data is lost between sessions without persistent storage.
  • Operations need role-based access so staff and manager can do different tasks.

2.3 Objectives

  1. Design a well-structured class hierarchy using inheritance and polymorphism.
  2. Implement all core OOP concepts: encapsulation, inheritance, polymorphism, abstraction, operator overloading, templates, and exception handling.
  3. Persist all data using C++ file handling (fstream).
  4. Provide a clean, menu-driven interface for staff and manager operations.
  5. Implement login-based access for two roles with one staff user and one manager user.

2.4 Scope

  • In Scope: Visitor management, ride management, event management, ticket booking, staff records, feedback, inventory, revenue report, file persistence.
  • Out of Scope: GUI interface, database connectivity, online booking, mobile app integration.

4. Object-Oriented Design

4.1 Class Hierarchy & UML Diagram

The application is built around the following class hierarchy:

<<abstract>> ParkEntity / | | | \ / | | | \ Visitor Ride Event Staff Inventory Ticket -- operator overloading / friend functions Feedback -- aggregation / average rating User -- authentication and roles ThemeParkSystem -- main controller class

4.2 Class Descriptions

ParkEntity (Abstract Base Class)

Abstract base class representing any park entity. Contains protected attributes: ID and name. Declares pure virtual methods display() and serialize() so every derived class provides its own implementation.

Visitor (Derived from ParkEntity)

Represents a park visitor. Adds age and email attributes and uses a static ID counter for automatic ID generation.

Ride (Derived from ParkEntity)

Represents a ride in the park. Stores price and operational status, and allows the system to show whether the ride is available or under maintenance.

Event (Derived from ParkEntity)

Represents a park event such as Eid Festival, Bake Sale, Chand Raat, Basant Kite Festival, and Holi Color Festival. Stores date, capacity, and number of registered visitors.

Staff (Derived from ParkEntity)

Represents staff members working in the park. Stores role and salary information.

Inventory (Derived from ParkEntity)

Represents park inventory items such as popcorn, ice cream, soft drinks, cotton candy, and merchandise.

Ticket

Stores ticket booking information and supports discount logic and output formatting.

Feedback

Stores visitor feedback and rating, and supports average rating calculation through a friend function.

User

Handles login credentials and role-based authentication for staff and manager users.

ThemeParkSystem

The main controller class that handles menus, login, file loading, file saving, and all operations of the application.

5. Implementation

5.1 OOP Concepts Implemented

OOP Concept Implementation Detail
Encapsulation All important data members are private and accessed through public methods.
Abstraction ParkEntity provides an abstract interface for all park records.
Inheritance Visitor, Ride, Event, Staff, and Inventory inherit from ParkEntity.
Polymorphism Virtual display() and serialize() functions are overridden in derived classes.
Abstract Classes ParkEntity cannot be instantiated directly because it has pure virtual functions.
Operator Overloading Ticket class overloads +, <, and << operators.
Templates readNumber() and findById() are template functions.
Exception Handling runtime_error and invalid_argument are used for invalid operations.
File Handling All data is saved and loaded from text files.
STL Usage vector, algorithm, numeric, and stringstream are used.
Friend Functions applyDiscount() and calculateAverageRating() access private data when needed.
Static Members Static nextId values generate unique IDs for each class.

5.2 Code Snippets

Abstract Base Class: ParkEntity

class ParkEntity { protected: int id; string name; public: ParkEntity(int id, const string &name) : id(id), name(name) {} virtual ~ParkEntity() {} int getId() const { return id; } string getName() const { return name; } virtual void display() const = 0; virtual string serialize() const = 0; };

Inheritance & Polymorphism: Visitor and Ride

class Visitor : public ParkEntity { private: int age; string email; static int nextId; public: Visitor(const string &name, int age, const string &email) : ParkEntity(nextId++, name), age(age), email(email) {} void display() const override { cout << "Visitor ID: " << id << " | Name: " << name << " | Age: " << age << " | Email: " << email << "\n" ; } }; class Ride : public ParkEntity { private: double price; bool operational; public: void display() const override { cout << "Ride ID: " << id << " | Name: " << name << " | Price: $" << fixed << setprecision(2) << price << " | Status: " << (operational ? "Operational" : "Maintenance" ) << "\n" ; } };

Operator Overloading: Ticket

class Ticket { private: int id; int visitorId; int rideId; double amount; public: double operator+(const Ticket &other) const { return amount + other.amount; } bool operator<(const Ticket &other) const { return amount < other.amount; } friend void applyDiscount(Ticket &ticket, double percentage); friend ostream &operator<<(ostream &os, const Ticket &ticket); };

File Handling

void saveUsers() const { ofstream uFile(usersFile); for (const auto &u : users) uFile << u.serialize() << "\n"; } void loadUsers() { ifstream file(usersFile); string line; while (getline(file, line)) { stringstream ss(line); string userIdStr, username, password, role; getline(ss, userIdStr, '|'); getline(ss, username, '|'); getline(ss, password, '|'); getline(ss, role, '|'); users.emplace_back(stoi(userIdStr), username, password, role); } }

9.1 Conclusion

The project gave both team members practical experience with class design, pointer-based polymorphism, file handling, and multi-class system development.
